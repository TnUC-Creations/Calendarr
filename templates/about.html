Changelog
v1.5.4 May 2026
  • Added additional security updates for release builds.
v1.5.3 May 2026
  • Fixed the Restart Service button so it is clickable again after service name handling changed.
v1.5.2 May 2026
  • Moved older changelog entries behind a Show older versions button so recent updates are easier to scan.
v1.5.1 May 2026
  • Rotated the built-in Google Calendar connection secret.
  • Added a Web UI Access setting for local-only or local-network access.
  • Added request protection for settings, cleanup, update, and other browser actions.
  • Added a backup warning because backups contain API keys and Google tokens.
  • In-app updates now require a matching SHA-256 checksum before installing.
v1.5.0 May 2026
  • Added day offsets for Radarr theater releases, Radarr digital releases, and Sonarr episodes, useful when upstream API dates are shifted by timezone conflicts.
  • Added Radarr tracking options so you can choose Theater Release, Digital Release, or both.
  • The Save Settings button and save status now stay visible while scrolling.
  • Settings now autosave after changes so leaving the page does not lose edits.
v1.4.0 May 2026
  • You can now sync to up to five Google calendars, with each calendar receiving Radarr releases, Sonarr episodes, or both.
  • Upcoming Releases and cleanup now include every configured calendar.
  • Radarr and Sonarr events can now use custom Google Calendar event colors per calendar.
  • Turning off a calendar source or removing a calendar target can now offer to clean up existing Calendarr events from that calendar.
v1.3.5 May 2026
  • Fixed all-day movie and episode events that could appear on the wrong day in Google Calendar.
v1.3.4 May 2026
  • Fixed sync so moved TV episodes are put back on the correct calendar date.
  • Fixed movie release dates that came from Radarr with a UTC timestamp showing on the wrong day.
v1.3.3 May 2026
  • Fixed TV episodes showing one day ahead in Google Calendar. Calendarr now uses the local broadcast date from Sonarr instead of converting from a UTC timestamp.
  • Fixed a missing log entry when the update checker could not reach GitHub.
v1.3.2 May 2026
  • You can now pick which Google Calendar Calendarr writes events to from the Settings page. Disconnect and reconnect your Google account once after updating to grant the new permission.
v1.3.1 May 2026
  • Narrowed the Google Calendar permission request to events only, instead of full calendar access. If you connected Google Calendar before this update, disconnect and reconnect once to apply the narrower permission.
v1.3.0 May 2026
  • System tray icon temporarily removed. The app now runs as a service only. Use the web UI at http://localhost:5000 to manage everything. The tray will return in a future release.
v1.2.2 May 2026
  • One-click update now works end to end. The update script stops the service, closes the tray app so the exe file is not locked, swaps in the new version, and restarts the service. The browser detects when the service is back and reloads the page automatically.
v1.2.1 May 2026
  • Radarr log now summarises movies with no upcoming release dates as a single count instead of one line per movie.
v1.2.0 May 2026
  • Sync logs now show exactly what happened at every step: which movies and shows were checked, which were skipped and why, what was added or removed, and whether Pushover sent a notification or why it didn't.
  • Fixed a bug where ignored shows were being removed from the calendar twice in the same sync, doubling the deleted count in logs and history.
  • Each sync log entry now includes the app version and service uptime, so you have context even if the log was recently cleared.
v1.1.0 May 2026
  • Backup now downloads a ZIP file containing both your settings and ignored shows list, so your full setup is captured in one file.
  • Restore accepts the ZIP backup and restores both your settings and ignored shows list in one go.
  • Ignored Shows page lists no longer expand the whole page for large libraries. Each box has a fixed height with its own scroll.
  • Active Shows list has a "Show ended" toggle in the header. Off by default so ended shows are hidden, keeping the list shorter. Your preference is saved between sessions.
v1.0.1 May 2026
  • Google Calendar connect now opens in a new tab so unsaved settings are not lost.
  • Test button errors are now capped at a short preview. Hover the badge to see the full message.
  • Save Settings button added at the top of the Settings page.
  • Light mode color fixes across the dashboard, settings, and ignored shows pages, including the Upcoming button and updated-event icon.
  • New app icon and logo added throughout the interface and browser tab.
  • Opening the Start Menu shortcut while Calendarr is already running now opens the web UI instead of starting a duplicate tray icon.
v1.0.0 May 2026
  • Google Calendar now connects via a one-click OAuth flow.
  • Dark and light mode toggle in the sidebar. Your choice is saved and restored on every page load.
  • Version number now shows in the sidebar next to the app name.
  • Preview Changes now includes auto-cleanup in the results. If auto-cleanup is enabled, the preview shows which past events would also be removed.
  • Ignored shows are now stored as JSON instead of a plain text file. Existing files migrate automatically on first launch.
  • Application errors, crashes, and HTTP failures are now written to the daily log file alongside sync output.
  • First beta release.
v0.9.1 April 2026
  • Preview Changes modal now has a Run Sync Now button that appears when there are pending changes, so you can go straight from preview to applying without closing and clicking again.
  • Last Run Changes on the dashboard now shows green plus, yellow arrows, and red minus icons matching the History page, instead of green checks for everything.
  • Reduced the gap between cards on the Settings page so the dark background shows through less between them.
v0.9.0 April 2026
  • Preview Changes button on the dashboard runs a full sync in read-only mode and shows exactly what would be added, updated, or removed without touching your calendar.
  • History page now has a live search bar and action filter buttons so you can quickly find specific events across 2000 entries.
  • Calendar events now include a description pulled from Radarr and Sonarr, so movie and episode synopses show up when you open an event in Google Calendar.
  • Pushover sound is now selectable in Settings from the full list of Pushover notification sounds.
  • Settings page layout rearranged: Radarr, Sonarr, and Google Calendar now sit side by side in equal columns, and Pushover has its own full-width row so it has room to breathe as more options are added.
v0.8.0 April 2026
  • Settings page now validates required fields before saving. Empty Calendar ID, API keys for enabled services, or Pushover credentials show an inline error and block the save rather than failing silently on the next sync.
  • Sonarr episode sync now loads all existing calendar events once before processing instead of making a separate API call for every episode. Large libraries will sync significantly faster.
  • If Radarr or Sonarr returns a malformed response, the sync now logs a visible error instead of silently completing with zero changes.
  • Auto-cleanup now matches events using the same naming patterns configured in your Event Naming Templates instead of hardcoded defaults. If you use custom event names, cleanup will correctly find and remove those events.
  • Favicon is now served from the embedded binary instead of the filesystem, so it works correctly regardless of where the exe is located.
v0.7.2 April 2026
  • Auto-cleanup now logs each deleted event by name instead of just the total count
  • Cleanup deletions now appear in History with a deleted badge, same as sync deletions
  • When auto-cleanup runs after a sync, the deleted count and event names show up in the dashboard status pills and Last Run Changes
  • Launching Calendarr from the Start Menu now starts the service automatically if it is not already running
  • Pushover notification settings now let you choose which event types trigger a notification. Added and updated are on by default, deleted is off.
  • Pushover now sends one notification per sync run instead of separate ones for sync and cleanup. Results are grouped by Added, Updated, and Removed with counts so the message is easier to read at a glance.
  • Last Run Changes on the dashboard now updates live without needing a page refresh
  • Fixed double-clicking a show on the Ignored Shows page not moving it between lists
  • Fixed auto-cleanup deleting today's events when the server clock is ahead in UTC. The cutoff now uses the server's local time.
